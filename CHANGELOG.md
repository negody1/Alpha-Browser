# Changelog

All notable user-facing changes to Alpha Browser.

## 0.1.1 — (prepared, not yet released)

### Alpha Proxy
- **Исправлен запуск прокси после перезапуска браузера** — устранены «осиротевшие»
  процессы, из-за которых PROXY-вкладки переставали работать после перезапуска.
- **Честные статусы подключения**: «Запускаем…» → «Проверяем соединение…» →
  «Готов к работе» — статус «готов» показывается только после реальной проверки
  выхода в интернет. Понятные сообщения об ошибках вместо технических.
- **Автовосстановление**: при сбое запуска прокси выполняется несколько повторных
  попыток; при удалении профиля доступ восстанавливается автоматически.
- **Раздел «Мой доступ»** в настройках: статус доступа и подключения, дата
  активации, версия профиля и браузера, кнопки «Проверить доступ» и
  «Проверить обновления», ссылка на страницу доступа.
- Фоновая проверка отзыва доступа; при отзыве — уведомление и остановка прокси.

### Onboarding и портал
- **Мастер первого запуска** и понятный онбординг (email → код → готово).
- **Полностью переработанный портал доступа** (страница заявки) и
  **админ-панель** в корпоративном стиле с фирменным логотипом и фоном Alpha.
- Первичная настройка администратора через веб-мастер (без паролей в окружении).

### Контент и приватность
- **Просмотр PDF прямо во вкладке** (масштаб, поиск, печать, скачивание,
  навигация по страницам).
- **Открытие локальных файлов** (PDF / изображения / текст) через `Ctrl+O`.
- **Очистка ссылок от трекинга**: автоматически удаляются `utm_*`, `fbclid`,
  `gclid`, `yclid`, `mc_eid` при переходах (параметры авторизации/оплаты не
  затрагиваются).
- Защита от утечки IP через WebRTC на PROXY-вкладках.

### Обновления
- Браузер проверяет новые версии на GitHub Releases и уведомляет о них; установка
  поверх текущей версии сохраняет настройки, профиль, историю и закладки.
